## Tek satırlık yorumlar

* Tek satırlık bir yorum `//` ile başlar.
* `//` ile satır sonu arasındaki her şey Dart derleyicisi tarafından yok sayılır.

```Dart
void main() {
  // Okuyarak Öğren
  print('Hello World!');
}
```

## Çok satırlı yorumlar

* Çok satırlı bir yorum `/*` ile başlar ve `*/` ile biter . `/*` ve `*/` arasındaki her şey Dart derleyicisi tarafından yok sayılır. Çok satırlı yorumlar iç içe geçebilir.

```
void main() {
/* 
  Bu bir çoklu yorum satırıdır.
  İstediğiniz kadar satır ekleyebilirsiniz!
  Dart bunu görmezden gelecektir 
*/
}
```

## Belgeleme yorumları

* Documentation comments, Dart'ta kod dokümantasyonu oluşturmak için kullanılan özel yorum satırlarıdır. Bu yorumlar, dart doc aracı tarafından otomatik olarak işlenerek güzel bir API dokümantasyonuna dönüştürülür.

* Documentation comments iki şekilde yazılabilir:
    - Tek satırlı format: ///
    - Çok satırlı format: /**  */

```
/// Bu bir tek satırlı documentation comment örneğidir
void birMethod() {}

/** 
  Bu ise çok satırlı
  documentation comment örneğidir
*/
void digerMethod() {}
```

:::note
Documentation comments içinde köşeli parantez [] kullanarak kodunuzdaki diğer elemanlara referans verebilirsiniz.
:::

### En İyi Uygulamalar

1. **Açıklayıcı Olun**

* Her public API elemanını dokümante edin
* Methodların ne yaptığını net bir şekilde açıklayın
* Parametrelerin ne işe yaradığını belirtin


2. **Format Kuralları**

* Her documentation comment bir tam cümle ile başlamalı
* Noktalama işaretlerine dikkat edin
* Paragraflar arasında boş satır bırakın


3. **Örnekler Ekleyin**
```
/// String'i tersine çevirir.
///
/// Örnek:
/// ```dart
/// print(reverseString('hello')); // Çıktı: 'olleh'
/// ```
String reverseString(String input) {
  return input.split('').reversed.join();
}
```

4. Özel Durumları Belirtin

* Methodun fırlatacağı exception'ları belirtin
* Null olabilecek değerleri işaretleyin
* Performans uyarılarını ekleyin

### Dart Doc Etiketleri

* @deprecated - Kullanımdan kaldırılmış API'ları işaretler
* @override - Override edilmiş methodları belirtir
* @required - Zorunlu parametreleri işaretler
* @todo - Yapılacak işleri belirtir

```
/// Bu eski bir method, bunun yerine [yeniMethod] kullanın.
@deprecated
void eskiMethod() {}
```

* İyi yazılmış documentation comments:

    - Kodunuzu daha anlaşılır kılar
    - Ekip içi iletişimi kolaylaştırır
    - Otomatik dokümantasyon oluşturmayı sağlar
    - Kodunuzun bakımını kolaylaştırır
