## Operatörler

- Dart, aşağıdaki tabloda gösterilen operatörleri destekler. Tablo, Dart'ın operatör ilişkilerinin bir yaklaşımı olan en yüksekten en düşüğe operatör ilişkisini ve operatör önceliğini gösterir. Bu operatörlerin çoğunu sınıf üyeleri olarak uygulayabilirsiniz .

### Operatör Tablosu

| Tanım                    | Operatör                                               | Birleşme |
| ------------------------ | ------------------------------------------------------ | -------- |
| unary postfix            | `expr++` `expr--` `()` `[]` `?[]` `.` `?.` `!`         | None     |
| unary prefix             | `-expr` `!expr` `-expr` `++expr` `--expr` `await expr` | None     |
| multiplicative           | `*` `/` `%` `~/`                                       | Left     |
| additive                 | `+` `-`                                                | Left     |
| shift                    | `<<` `>>` `>>>`                                        | Left     |
| bitwise AND              | `&`                                                    | Left     |
| bitwise XOR              | `^`                                                    | Left     |
| bitwise OR               | `\|`                                                   | Left     |
| relational and type test | `>=` `>` `<=` `<` `as` `is` `is!`                      | None     |
| equality                 | `==` `!=`                                              | None     |
| logical AND              | `&&`                                                   | Left     |
| logical OR               | `\|\|`                                                 | Left     |
| if-null                  | `??`                                                   | Left     |
| conditional              | `expr1 ? expr2 : expr3`                                | Right    |
| cascade                  | `..` `?..`                                             | Left     |
| assignment               | `=` `*=` `/=` `+=` `-=` `&=` `^=` etc.                 | Right    |
| spread                   | `...` `...?`                                           | None     |

<Alert type="warning">
  Önceki tablo yalnızca yardımcı bir kılavuz olarak kullanılmalıdır. Operatör önceliği ve
  ilişkisellik kavramı, dil gramerinde bulunan gerçeğin bir yaklaşımıdır.
</Alert>

- Operatörleri kullandığınızda ifadeler yaratırsınız. İşte operatör ifadelerinin bazı örnekleri:

```Dart
a++
a + b
a = b
a == b
c ? a : b
a is T
```

## Operatör önceliği örneği

- Operatör tablosunda , her operatör onu izleyen satırlardaki operatörlerden daha yüksek önceliğe sahiptir. Örneğin, çarpma operatörü, mantıksal AND operatöründen daha yüksek önceliğe sahip olan %eşitlik operatöründen daha yüksek önceliğe sahiptir (ve dolayısıyla daha önce yürütülür) . Bu öncelik, aşağıdaki iki satır kodun aynı şekilde yürütüldüğü anlamına gelir:`==&&`

```Dart
// Parantezler okunabilirliği artırır.
if ((n % i == 0) && (d % i == 0)) ...

// Okunması daha zor ama eşdeğer.
if (n % i == 0 && d % i == 0) ...
```

<Alert type="warning">
  İki işlenen alan operatörler için, en soldaki işlenen hangi yöntemin kullanılacağını belirler.
  Örneğin, bir `Vector` nesneniz ve bir `Point` nesneniz varsa, `aVector` + `aPoint` `Vector`
  toplamasını `(+)` kullanır.
</Alert>

## Aritmetic operatörler

- Dart, aşağıdaki tabloda gösterildiği gibi, olağan aritmetik operatörleri destekler.

- Örnek

```Dart
assert(2 + 3 == 5);
assert(2 - 3 == -1);
assert(2 * 3 == 6);
assert(5 / 2 == 2.5); // Result is a double
assert(5 ~/ 2 == 2); // Result is an int
assert(5 % 2 == 1); // Remainder

assert('5/2 = ${5 ~/ 2} r ${5 % 2}' == '5/2 = 2 r 1');
```

<Alert type="note">
Dart ayrıca hem prefix hem de postfix artırma ve azaltma operatörlerini destekler.
</Alert>

## Artırma ve Azaltma Operatörleri Tablosu

| Operatör | Anlam                                    |
| -------- | ---------------------------------------- |
| `++var`  | `var = var + 1` (ifade değeri `var + 1`) |
| `var++`  | `var = var + 1` (ifade değeri `var`)     |
| `--var`  | `var = var - 1` (ifade değeri `var - 1`) |
| `var--`  | `var = var - 1` (ifade değeri `var`)     |

### Kullanım Örneği

```dart
void main() {
 var a = 5;

 print(++a); // Çıktı: 6 (önce artırır, sonra değeri döndürür)
 print(a);   // Çıktı: 6

 var b = 5;
 print(b++); // Çıktı: 5 (önce değeri döndürür, sonra artırır)
 print(b);   // Çıktı: 6

 var c = 5;
 print(--c); // Çıktı: 4 (önce azaltır, sonra değeri döndürür)
 print(c);   // Çıktı: 4

 var d = 5;
 print(d--); // Çıktı: 5 (önce değeri döndürür, sonra azaltır)
 print(d);   // Çıktı: 4
}
```

## Eşitlik ve ilişkisel operatörler

### Karşılaştırma Operatörleri Tablosu

| Operatör | Anlam                            |
| -------- | -------------------------------- |
| `==`     | Eşit; aşağıdaki tartışmaya bakın |
| `!=`     | Eşit değil                       |
| `>`      | Daha büyük                       |
| `<`      | Daha az                          |
| `>=`     | Büyük veya eşit                  |
| `<=`     | Küçük veya eşit                  |

### Kullanım Örneği

```dart
void main() {
 var a = 5;
 var b = 3;

 print(a == b); // Çıktı: false
 print(a != b); // Çıktı: true
 print(a > b);  // Çıktı: true
 print(a < b);  // Çıktı: false
 print(a >= b); // Çıktı: true
 print(a <= b); // Çıktı: false
}
```

## Nesne Karşılaştırma ve `==` Operatörü

İki nesnenin (x ve y) aynı şeyi temsil edip etmediğini test etmek için `==` operatörü kullanılır. (Eğer iki nesnenin tamamen aynı nesne olup olmadığını kontrol etmeniz gereken nadir durumlarda `identical()` fonksiyonunu kullanın.)

### `==` Operatörü Nasıl Çalışır?

1. Eğer `x` veya `y`'den biri `null` ise:

   - Her ikisi de `null` ise `true` döner
   - Sadece biri `null` ise `false` döner

2. Diğer durumlarda, `x` nesnesinin `==` metodunu `y` argümanıyla çağırarak sonucu döndürür.

<Alert type="note">
`==` gibi operatörler aslında ilk operandları üzerinde çağrılan metodlardır.
</Alert>

### Örnek Kullanım

```dart
void main() {
  // Eşitlik kontrolü
  assert(2 == 2);    // true - sayılar eşit
  assert(2 != 3);    // true - sayılar eşit değil

  // Büyüklük-küçüklük karşılaştırması
  assert(3 > 2);     // true - 3, 2'den büyük
  assert(2 < 3);     // true - 2, 3'ten küçük
  assert(3 >= 3);    // true - 3, 3'e eşit veya büyük
  assert(2 <= 3);    // true - 2, 3'ten küçük veya eşit

  // Null karşılaştırma örnekleri
  String? a = null;
  String? b = null;
  String c = "test";

  assert(a == b);    // true - iki null değer eşittir
  assert(a != c);    // true - null ile null olmayan eşit değildir
}
```

## Tip Test Operatörleri Tablosu

| Operatör | Anlam                                                          |
| -------- | -------------------------------------------------------------- |
| `as`     | Typecast (ayrıca `library prefixes` belirtmek için kullanılır) |
| `is`     | Nesne belirtilen tipe sahipse true döner                       |
| `is!`    | Nesne belirtilen tipe sahip değilse true döner                 |

### Tip Kontrolü ve Runtime'da Type Testing

`as`, `is`, ve `is!` operatörleri, runtime sırasında tip kontrolü yapmak için kullanışlı araçlardır. Bir nesnenin (`obj`) belirli bir tipi (`T`) uygulayıp uygulamadığını kontrol etmek için kullanılırlar.

Örneğin, `obj is Object?` her zaman true değerini döndürür çünkü her nesne `Object`'ten türer.

### `as` Operatörü Kullanımı

`as` operatörünü yalnızca bir nesnenin belirli bir tipte olduğundan kesinlikle eminseniz kullanın.

```dart
void main() {
 // DİKKAT: Bu kullanım risklidir
 (employee as Person).firstName = 'Bob';
}
```

### `is` Operatörü Kullanımı

- Nesnenin tipinden emin değilseniz, nesneyi kullanmadan önce is ile tip kontrolü yapın.

```Dart
void main() {
  // DOĞRU YAKLAŞIM: Önce tip kontrolü yap
  if (employee is Person) {
    employee.firstName = 'Bob';  // Güvenli kullanım
  }
}
```

<Alert type="warning">
Yukarıdaki iki kod örneği birbirine eşdeğer değildir:

İlk örnekte (as kullanımı), eğer employee null ise veya Person tipinde değilse, program exception fırlatır.
İkinci örnekte (is kullanımı), eğer koşul sağlanmazsa blok içine girmez ve herhangi bir hata oluşmaz.
</Alert>

<Alert type="tip">
Kullanım Önerileri

Tip dönüşümlerinde öncelikle is ile kontrol yapın
as operatörünü sadece tipten kesinlikle emin olduğunuz durumlarda kullanın
Null olabilecek değişkenlerle çalışırken ekstra dikkatli olun
is! operatörünü bir tipin olmaması durumunu kontrol etmek için kullanın
</Alert>

```Dart
void main() {
  dynamic value = 42;

  // is kullanımı
  if (value is int) {
    print('Bu bir tam sayıdır');
  }

  // is! kullanımı
  if (value is! String) {
    print('Bu bir string değildir');
  }

  // as kullanımı (dikkatli kullanılmalı)
  try {
    String text = value as String;  // Runtime hatası!
  } catch (e) {
    print('Tip dönüşümü başarısız: $e');
  }
}
```

## Atama operatörleri

- Daha önce gördüğünüz gibi, `=` operatör kullanarak değerler atayabilirsiniz. Yalnızca atanmış değişken `null` olduğunda atama yapmak için `??=` operatörü kullanın.

```Dart
// a ya bir değer atayın
a = value;
// b boşsa b'ye değer ata; aksi takdirde b aynı kalır
b ??= value;
```

- `+=` gibi bileşik atama operatörleri bir işlemi bir atamayla birleştirir.

## Bileşik Atama (Compound Assignment) Operatörleri

`+=` gibi bileşik atama operatörleri, bir operasyonu atama işlemiyle birleştirir.

### Mevcut Operatörler

| Birinci Grup              | İkinci Grup                | Üçüncü Grup      |
| ------------------------- | -------------------------- | ---------------- |
| `=` `*=` `%=` `>>>=` `^=` | `+=` `/=` `<<=` `&=` `\|=` | `-=` `~/=` `>>=` |

### Operatörlerin Çalışma Mantığı

Bir `op` operatörü için:

```dart
a op= b
```

- ifadesi, aşağıdaki ile aynı anlama gelir:

```Dart
a = a op b
```

### Örnek Açıklama

- Eğer `+=` operatörünü ele alırsak:

```Dart
a += b
// ifadesi şuna eşdeğerdir:
a = a + b
```

### Pratik Kullanım Örneği

```Dart
void main() {
  // Temel atama operatörü ile değer atama
  var a = 2;  // = operatörü ile başlangıç değeri atama

  // Bileşik atama operatörü kullanımı
  a *= 3;     // Çarpma ve atama: a = a * 3

  // Sonucu kontrol etme
  assert(a == 6);  // true, çünkü 2 * 3 = 6

  // Diğer bileşik atama örnekleri
  var b = 10;
  b += 5;     // b = 15 (toplama ve atama)
  b -= 3;     // b = 12 (çıkarma ve atama)
  b ~/= 2;    // b = 6  (tam sayı bölme ve atama)
  b %= 4;     // b = 2  (mod  ve atama)
}
```

<Alert type="tip">

1. Bileşik atama operatörleri kod yazımını daha kısa ve okunabilir hale getirir
2. Her aritmetik veya bitsel operatörün bir bileşik atama karşılığı vardır
3. Operatörler soldan sağa doğru değerlendirilir
4. İşlem ve atama tek bir adımda gerçekleşir
5. Bileşik atama operatörleri özellikle döngülerde ve matematik işlemlerinde kod tekrarını azaltmak için çok kullanışlıdır.
</Alert>

## Mantıksal operatörler

- Mantıksal operatörleri kullanarak Boole ifadelerini tersine çevirebilir veya birleştirebilirsiniz.

### Mantıksal Operatörler Tablosu

| Operatör | Anlam                                                         |
| -------- | ------------------------------------------------------------- | --- | -------------- |
| `!expr`  | aşağıdaki ifadeyi tersine çevirir (false'ı true'ya, ve tersi) |
| `        |                                                               | `   | mantıksal VEYA |
| `&&`     | mantıksal VE                                                  |

### Kullanım Örneği

```dart
void main() {
 bool a = true;
 bool b = false;

 // ! operatörü
 print(!a);      // Çıktı: false (true'nun tersi)
 print(!b);      // Çıktı: true  (false'un tersi)

 // || operatörü (VEYA)
 print(a || b);  // Çıktı: true  (en az biri true)
 print(b || b);  // Çıktı: false (ikisi de false)

 // && operatörü (VE)
 print(a && b);  // Çıktı: false (ikisi birden true değil)
 print(a && a);  // Çıktı: true  (ikisi de true)
}
```

<Alert type="note">

1. ! operatörü tekli (unary) bir operatördür ve sadece boolean değerleri tersine çevirir
2. || operatöründe ifadelerden birinin true olması yeterlidir
3. && operatöründe tüm ifadelerin true olması gerekir
4. && operatörü || operatöründen daha yüksek önceliğe sahiptir
</Alert >

- Mantıksal operatörler genellikle koşul ifadelerinde ve kontrol akışlarında kullanılır:

```Dart
if (!isLoading && (isAdmin || hasPermission)) {
  // İşlemler burada
}
```

## Koşullu ifadeler

- Dart, if-else ifadelerinin yerine kullanılabilecek iki özel operatör sunar:

### 1. Koşul (Ternary) Operatörü: `condition ? expr1 : expr2`

```dart
if ? ifade1 : ifade2
```

- Eğer koşul true ise, ifade1 değerlendirilir ve sonucu döndürülür
- Eğer koşul false ise, ifade2 değerlendirilir ve sonucu döndürülür

```Dart
void main() {
  var isPublic = true;
  var visibility = isPublic ? 'public' : 'private';
  print(visibility); // Çıktı: 'public'
}
```

### Null-Coalescing (If-Null) Operatörü: expr1 ?? expr2

```
ifade1 ?? ifade2
```

- Eğer ifade1 null değilse, değeri döndürülür
- Eğer ifade1 null ise, ifade2 değerlendirilir ve sonucu döndürülür

```Dart
void main() {
  String? name;
  print(name ?? 'Misafir'); // Çıktı: 'Misafir'

  name = 'Ahmet';
  print(name ?? 'Misafir'); // Çıktı: 'Ahmet'
}
```

<Alert type="note">

1. ?? operatörü özellikle null kontrolü gerektiren durumlarda kodu daha okunabilir hale getirir
2. Ternary operatör (?:) basit if-else ifadelerini tek satırda yazmamızı sağlar
3. Her iki operatör de kodun daha kısa ve öz olmasını sağlar
4. Bu operatörler iç içe kullanılabilir, ancak okunabilirlik için dikkatli olunmalıdır
</Alert>

<Alert type="warning">
İç içe fazla koşul operatörü kullanmak kodun okunabilirliğini azaltabilir. Böyle durumlarda klasik if-else yapısını tercih etmek daha iyi olabilir.
</Alert>
